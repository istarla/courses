# HTML Forms - Part 2

## More Form Elements


Let's explore more essential elements used to build versatile and
user-friendly forms.

### The `<textarea>` Element

Used for multi-line text input, suitable for longer messages, comments,
or descriptions. Unlike `<input>`, it's not an empty
element; any text between `<textarea>` and
`</textarea>` becomes its default content.

* **`name`:** Specifies the name sent to the server.
* **`id`:** For associating labels.
* **`rows`:** Suggests the visible number of text lines (height).
* **`cols`:** Suggests the visible width in average character widths. (Note: CSS
  is generally preferred for controlling exact dimensions).
* **`required`:** Makes input mandatory.
* **`placeholder`:** Provides a hint.

*Sample Code*
```html
<label for="comment_box">Your Comment:</label>
<textarea id="comment_box" name="comment" rows="5" cols="40" placeholder="Enter your feedback here..." required></textarea>
```

### The `<select>` Element (Dropdown Lists)

Creates a dropdown list allowing users to choose one or more options
from a predefined list.

* **`<select>`:** The container element. Requires `name` and
  `id` attributes.
  + **`multiple`:** Boolean attribute allowing selection of
    multiple options (usually requires holding Ctrl/Cmd while
    clicking).
  + **`size`:** Specifies the number of visible options in the
    list (creates a scroll box instead of a dropdown if > 1).
* **`<option>`:** Defines an individual item within the dropdown list.
  + **`value`:** The value sent to the server if this option is
    selected. **Crucial attribute.** If omitted, the text
    content of the option is sent.
  + **`selected`:** Boolean attribute to pre-select this option
    by default.
  + The text content between `<option>` and
    `</option>` is what the user sees in the list.
* **`<optgroup>`:** Used to group related options within a dropdown list. Requires a
  `label` attribute which provides the heading for the group
  (this heading is not selectable).

*Sample Code*
```html
<label for="country_select">Choose your country:</label>
<select id="country_select" name="country">
  <option value="">--Please choose an option--</option> <!-- Placeholder option -->
  <optgroup label="North America">
    <option value="US">United States</option>
    <option value="CA">Canada</option>
    <option value="MX">Mexico</option>
  </optgroup>
  <optgroup label="Europe">
    <option value="GB" selected>United Kingdom</option> <!-- Pre-selected -->
    <option value="DE">Germany</option>
    <option value="FR">France</option>
  </optgroup>
</select>
```

### The `<button>` Element

Offers more flexibility than `<input type="button">`,
`<input type="submit">`, or
`<input type="reset">` because you can include HTML
content (like images or text formatting) inside it.

* **`type`:** Specifies the button's behavior:
  + **`submit`**(Default): Submits the form
    data. Acts like `<input type="submit">`.
  + **`reset`:** Resets the form controls. Acts like
    `<input type="reset">`.
  + **`button`:** A clickable button with no default behavior. Typically used
    with JavaScript to trigger custom actions.
* **`name` / `value`:** Can be included. If a submit button has a `name` and
  `value`, these are sent with the form data, which can be
  useful if you have multiple submit buttons and need to know which one
  was clicked.

*Sample Code*
```html
<button type="submit" name="action" value="save">
  <!-- You can add icons/images here -->
  <img src="save-icon.png" alt="" width="16" height="16"> <strong>Save</strong> Draft
</button>

<button type="reset">Clear All</button>

<button type="button" onclick="alert('Hello!');">Click Me (JS)</button>
```

### The `<fieldset>` and `<legend>` Elements

Used to group related controls within a form, visually and semantically.

* **`<fieldset>`:** Draws a box (border) around the grouped elements. Helps organize
  complex forms.
* **`<legend>`:** Provides a caption or title for the `<fieldset>`.
  It should be the first child of the `<fieldset>`.
  Improves accessibility by clearly labeling the group.

*Sample Code*
```html
<fieldset>
  <legend>Contact Information</legend>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email"><br>

  <label for="phone">Phone:</label>
  <input type="tel" id="phone" name="phone"> <!-- type="tel" is another input type -->
</fieldset>
```

## Other Useful Input Types (HTML5)


HTML5 introduced several new input types that provide better user
experiences and built-in browser validation:

* **`type="number"`:** For numerical input. Browsers often display spinner controls
  (up/down arrows). Attributes like `min`, `max`,
  and `step` can be used for constraints.
* **`type="date"`**, **`type="month"`**, **`type="week"`**, **`type="time"`**, **`type="datetime-local"`**: Provide specialized controls for picking dates and times, often
  showing a calendar or time picker interface. Browser support and
  appearance vary.
* **`type="range"`:** Displays as a slider control for selecting a value within a defined
  range (using `min`, `max`,
  `step` attributes).
* **`type="color"`:** Provides a color picker interface.
* **`type="url"`:** For web addresses. Provides validation for URL format.
* **`type="tel"`:** For telephone numbers. Doesn't enforce a strict format (as formats
  vary widely) but can trigger numeric keypads on mobile devices.
* **`type="search"`:** Functionally similar to `text`, but browsers might style
  it differently (e.g., adding a clear 'x' icon) and it indicates
  semantic purpose.
