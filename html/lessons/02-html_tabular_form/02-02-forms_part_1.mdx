# HTML Forms - Part 1

## Introduction to Forms
<section id="introduction-to-forms" />

HTML forms are fundamental for user interaction on the web. They allow
users to input data, which can then be sent to a web server for
processing (like logging in, signing up, submitting comments, making
purchases, etc.).

Creating a form involves two main parts:

1. The HTML structure of the form itself (using form elements).
2. A server-side script (written in languages like PHP, Python, Node.js,
   etc.) to process the submitted data.
   **This course focuses only on the HTML part.** We won't
   cover server-side processing.

## The `<form>` Element
<section id="the-form-element" />

All form controls must be contained within a
`<form>` element. This element acts as
a container and defines how the form data should be submitted.

Key attributes of the `<form>` tag:

* `action`: Specifies the URL of the server-side script that will process the
  form data. If omitted, the data is typically sent to the same page.
* `method`: Specifies the HTTP method to use when submitting the form data.
  Common values:
  + `get` (Default): Appends the form data
    to the URL specified in the `action` attribute (e.g.,
    `process.php?name=John&email=...`). Useful for simple,
    non-sensitive data like search queries. Data is visible in the URL
    and has length limitations.
  + `post`: Sends the form data in the HTTP request body. More secure for
    sensitive information (like passwords) as the data is not visible
    in the URL. No data length limitations. Preferred for forms that
    modify data on the server.
* `name`: A name for the form, useful if you need to reference it with
  JavaScript. (Optional but can be helpful).

*Sample Code*
```html
<form action="/submit-handler" method="post">
  <!-- Form elements go here -->
</form>
```

## The `<input>` Element
<section id="the-input-element" />

The `<input>` element is the most
versatile form element. It's an empty element (no closing tag) and its
behavior is determined by its
`type` attribute.

Essential attributes for most `<input>` types:

* `type`: Defines the type of input control (text field, password, checkbox,
  radio button, submit button, etc.).
* `name`: Provides a name for the input field. This name is sent to the
  server along with the input's value when the form is submitted.
  **Crucial for data processing.**
* `id`: Provides a unique identifier for the input element within the
  document. Used primarily for associating labels
  (`<label>`) with the input and for JavaScript/CSS
  manipulation.
* `value`: Specifies the initial value of the input field. For checkboxes and
  radio buttons, it defines the value sent to the server if that option
  is selected. For text fields, it's the default text. For submit/reset
  buttons, it's the text displayed on the button.
* `placeholder`: Provides a short hint (e.g., sample value or description) displayed
  in the input field before the user enters a value. Disappears when the
  user starts typing.
* `required`: A boolean attribute indicating that the user must fill in a value
  before submitting the form.

### Common `<input>` Types (Part 1):

* **`type="text"`:** A single-line text input field.

  ```html
  <input type="text" id="username" name="username" placeholder="Enter your username" required>
  ```
  <input type="text" id="username" name="username" placeholder="Enter your username" required style={{border:"1px solid #ccc"}} />

* **`type="password"`:** Similar to "text", but the characters entered are obscured (e.g.,
  shown as asterisks or dots).

  ```html
  <input type="password" id="pwd" name="password" required>
  ```
  <input type="password" id="pwd" name="password" required style={{border:"1px solid #ccc"}} />

* **`type="email"`:** Designed for email addresses. Browsers may provide validation to
  check if the entered text looks like a valid email format. On mobile
  devices, it might trigger a keyboard optimized for email input.

  ```html
  <input type="email" id="user_email" name="email" placeholder="you@example.com">
  ```
  <input type="email" id="user_email" name="email" placeholder="you@example.com" style={{border:"1px solid #ccc"}} />

* **`type="submit"`:** A button that, when clicked, submits the form data to the URL
  specified in the form's `action` attribute using the
  specified `method`. The text on the button comes from its
  `value` attribute (defaults to "Submit" or similar).

  ```html
  <input type="submit" value="Sign Up" style="border : 1px solid blue; background-color: blue; color:white; border-radius: 4px; padding: 4px;">
  ```
  <input type="submit" value="Sign Up" style={{border:"1px solid blue", backgroundColor:"blue", color: "white", borderRadius:"4px", padding:"4px", cursor:"pointer"}} />

* **`type="reset"`:** A button that, when clicked, resets all form controls within the
  form to their initial values.

  ```html
  <input type="reset" value="Clear Form">
  ```

* **`type="radio"`:** A radio button. Allows the user to select only one option from a
  group of choices. All radio buttons in a group must share the same
  `name` attribute but have different
  `value` attributes. Use `<label>` tags for
  usability.

  ```html
  <input type="radio" id="option1" name="fav_color" value="red">
  <label for="option1">Red</label><br>
  <input type="radio" id="option2" name="fav_color" value="blue" checked> <!-- 'checked' pre-selects this option -->
  <label for="option2">Blue</label>
  ```
  **Output:**
  <>
  <br/>
    <input type="radio" id="option1" name="fav_color" value="red" />
    <label htmlFor="option1">Red</label>
    <br />
    <input
      type="radio"
      id="option2"
      name="fav_color"
      value="blue"
      defaultChecked
    />
    <label htmlFor="option2">Blue</label>
  </>


* **`type="checkbox"`:** A checkbox. Allows the user to select zero or more options from a
  group. Checkboxes representing different choices should have unique
  `name` attributes (unless they are part of a group where
  multiple values with the same name are expected, common in server-side
  processing). Use `<label>` tags.

  ```html
  <input type="checkbox" id="terms" name="agree_terms" value="yes" required>
  <label for="terms">I agree to the terms</label><br>
  <input type="checkbox" id="newsletter" name="subscribe_news" value="true" checked>
  <label for="newsletter">Subscribe to newsletter</label>
  ```
  **Output**:
  <br/>
  <input type="checkbox" id="terms" name="agree_terms" value="yes" required />
    <label for="terms">I agree to the terms</label><br/>
    <input type="checkbox" id="newsletter" name="subscribe_news" value="true" checked />
    <label for="newsletter">Subscribe to newsletter</label>
## The `<label>` Element
<section id="the-label-element" />

The `<label>` element provides a
caption or label for a form control. It improves usability and
accessibility:

* Clicking on the label focuses or activates the associated control
  (e.g., clicking the label for a text input puts the cursor in the
  input; clicking the label for a radio button selects that button).
* Screen readers announce the label when the user focuses on the
  associated form control.

Associate a label with a control using the
`for` attribute. The value of the
`for` attribute must match the
`id` attribute of the corresponding form
control.

*Sample Code*
```html
<label for="user_name">Username:</label>
<input type="text" id="user_name" name="username">
```
**Output:**

<label for="user_name">Username:</label>
<input type="text" id="user_name" name="username" style={{border:"1px solid blue"}} />

Alternatively, you can wrap the input element inside the label (implicit
association), though the explicit `for`/`id`
method is often considered more robust:

*Sample Code*
```html
<label>Username:
  <input type="text" name="username"> <!-- ID is still recommended for consistency -->
</label>
```
**Output:**
<label>
Username:<input type="text" name="username" style={{border:"1px solid blue"}}/> 
 </label> 
{/* ID is still recommended for consistency */}
