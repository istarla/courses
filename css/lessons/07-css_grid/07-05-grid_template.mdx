# Grid Template Areas and Shorthand

Yesterday we learned how to place grid items using line numbers. Today,
we'll explore a more visual and often more intuitive method: using named
grid areas with the grid-template-areas property. We will
also look at the grid shorthand property.

## 1. Defining Named Areas: grid-template-areas

The grid-template-areas property allows you to define a
template for your grid layout by assigning names to grid cells. You
create a visual representation of the grid structure directly in your
CSS.

**Syntax:**

* You provide strings, where each string represents a row.
* Within each string, you list space-separated names that correspond to
  the columns in that row.
* The same name repeated in adjacent cells creates a single grid area
  spanning those cells.
* A period (`.`) or multiple periods without spaces
  represents an empty cell (no assigned area name).
* The number of names per string defines the number of columns, and the
  number of strings defines the number of rows. All strings must have
  the same number of columns.

**Example: Defining a Layout with Named Areas**

        <div className="area-container">
          <div className="area-item area-header">Header</div>
          <div className="area-item area-sidebar">Sidebar</div>
          <div className="area-item area-main">Main Content</div>
          <div className="area-item area-footer">Footer</div>
        </div>

```css
.area-container {
  display: grid;
  /* Visualize the grid layout: */
  grid-template-areas:
    "header  header   header"   /* Row 1 */
    "sidebar main     main"     /* Row 2 */
    "sidebar content2 ."        /* Row 3 (example with empty cell) */
    "footer  footer   footer";   /* Row 4 */

  /* Define track sizes (highly recommended when using areas) */
  grid-template-columns: 150px 1fr 1fr;
  grid-template-rows: auto 1fr 1fr auto; /* Header/Footer auto, main rows flexible */
  gap: 10px;
}
```

In this example, we've defined a grid visually. The 'header' area spans
all three columns in the first row, 'sidebar' spans the first column
across rows 2 and 3, 'main' spans columns 2 and 3 in row 2, 'content2'
occupies a single cell, and an empty cell is defined with
`.`.

## 2. Assigning Items to Areas: `grid-area`

Once you've defined named areas using `grid-template-areas`,
you assign grid items to those areas using the
`grid-area` property on the item itself.

**Syntax:** `grid-area: <area-name>;`

The `<area-name>` must match one of the names defined
in the `grid-template-areas` property of the container.

```css
.area-header {
  grid-area: header; /* Assigns this element to the 'header' grid area */
  background-color: lightblue;
}
.area-sidebar {
  grid-area: sidebar; /* Assigns this element to the 'sidebar' grid area */
  background-color: lightgreen;
}
.area-main {
  grid-area: main; /* Assigns this element to the 'main' grid area */
  background-color: lightcoral;
}
.area-content2 {
  grid-area: content2;
  background-color: lightyellow;
}
.area-footer {
  grid-area: footer; /* Assigns this element to the 'footer' grid area */
  background-color: lightgray;
}
```

Using `grid-template-areas` offers a very readable and
maintainable way to structure complex layouts. It's often easier to
understand and modify than complex line-number placements. Remember that
`grid-area` is also the shorthand for line-based placement
(`row-start / col-start / row-end / col-end`), so it serves a
dual purpose.

## 3. The grid Shorthand Property

The `grid` property is a complex shorthand that can define
multiple grid properties in one declaration, including explicit tracks
(rows/columns), implicit tracks, gaps, and template areas.

Its syntax can be quite involved, but a common and useful form allows
defining `grid-template-rows`,
`grid-template-columns`, and
`grid-template-areas` together.

**Syntax (for rows, columns, areas):**
`grid: <grid-template-rows> /
<grid-template-columns>;`
(This doesn't set areas).

**Syntax (including areas):** You define rows, including
their names (optional, in square brackets), area names within strings,
and row sizes, followed by a forward slash `/` and the column
track definitions.

```css
/* Example combining areas, row sizing, line names, and column sizing */
.shorthand-container {
  display: grid;
  gap: 5px;
  grid:
    [row1-start] "header  header"  auto        [row1-end] /* Row 1: name, areas, size, name */
    [row2-start] "sidebar main"    1fr         [row2-end] /* Row 2: name, areas, size, name */
    [row3-start] "footer  footer"  min-content [row3-end] /* Row 3: name, areas, size, name */
    /            [col1-start] 120px [col1-end] 1fr [col2-end]; /* Column defs after slash */
    /* Defines:
       - grid-template-areas: "header header" "sidebar main" "footer footer"
       - grid-template-rows: auto 1fr min-content
       - grid-template-columns: 120px 1fr
       - Row line names: row1-start, row1-end, row2-start, etc.
       - Column line names: col1-start, col1-end, col2-end */
}

/* Corresponding item assignments */
.sh-header { grid-area: header; }
.sh-sidebar { grid-area: sidebar; }
.sh-main { grid-area: main; }
.sh-footer { grid-area: footer; }
```

Example using `grid` shorthand:

        <div className="shorthand-container">
          <div className="sh-item sh-head">Header</div>
          <div className="sh-item sh-nav">Nav</div>
          <div className="sh-item sh-main">Main</div>
          <div className="sh-item sh-foot">Footer</div>
        </div>

The `grid` shorthand is powerful but can be complex to read
and write. While it can condense code, sometimes using the individual
properties (`grid-template-columns`,
`grid-template-rows`, `grid-template-areas`,
`gap`) can be clearer, especially for intricate layouts.
Choose the approach that offers the best readability and maintainability
for your specific situation.
